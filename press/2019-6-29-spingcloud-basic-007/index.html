# 概念

在分布式系统中，由于服务数量巨多，为了方便服务配置文件统一管理，所以需要分布式配置中心组件。

本篇涉及项目的结构为一个Config Server用于链接Git仓库，一个Config Client通过server来展示配置文件数据。 同时使用两个bus来模拟配置文件刷新。

# Config Server

新建一个Config Server，分别配置pom.xml、application.yml，并在启动类中添加注释@EnableConfigServer。

		<dependency>
			<groupId>org.springframework.cloud</groupId>
			<artifactId>spring-cloud-config-server</artifactId>
		</dependency>
server:
  port: 9080
spring:
  application:
    name: springcloud-config-server
  cloud:
    config:
      server:
        git:
          uri: https://github.com/fymod/easy-spring-cloud
          username:
          password:

以上步骤完成后，直接启动即可，然后浏览器地址栏输入http://localhost:9080/springcloud-7-config-dev.properties 查看数据，会显示对应的文件内容。浏览器输入 http://localhost:9080/springcloud-config-server/springcloud-7-config-dev 查看数据，会显示对应文件详细信息。

http请求地址：/{application}/{profile}[/{label}] 资源文件： /{application}-{profile}.yml /{label}/{application}-{profile}.yml /{application}-{profile}.properties /{label}/{application}-{profile}.properties 示例：本例中资源文件是config-dev.properties，其中config就是application，dev就是profile，匹配资源文件的第三种，访问地址就是/config(application)/dev(profile)

# BUS

以上demo的问题是，如果我修改了git仓库中文件的内容，项目并不会刷新。本节介绍如果应用刷新项目并通知到其他微服务。Config Server继续使用刚刚创建的。

在创建项目前，需要先安装RabbitMQ，安装步骤可点击这里 查看。

新建两个Config Client，分别配置pom.xml、application.yml、bootstrap.yml，然后提供对外调用接口的Java类。除了端口号设置成不一致外，其他的完全一样。

		</dependency>
		<dependency>
			<groupId>org.springframework.cloud</groupId>

Java类和其他常规不同的是需要增加注解@RefreshScope，这样在刷新之后才会将对应值改变。

import org.springframework.beans.factory.annotation.Value;
import org.springframework.cloud.context.config.annotation.RefreshScope;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
@RefreshScope
public class TestController {

	@Value("${profile}")
	private String profile;
	
	@GetMapping("/profile")
	public String getProfile() {
		return this.profile;
	}
	
}

# 结合Eureka

spring.application.name=config
spring.cloud.config.label=master
spring.cloud.config.profile=dev
#spring.cloud.config.uri= http://localhost:9080/
server.port=9081
eureka.client.serviceUrl.defaultZone=http://localhost:9080/eureka/
